# v0.3 Layered Persona Architecture — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Update eidola from v0.2 to v0.3 — layered persona architecture with portrait/, separate memory database, and session lifecycle hooks.

**Architecture:** Three-domain split (arkiv/, portrait/, memory/) with cognitive hierarchy. CLAUDE.md is behavioral core, portrait/ holds synthesized narrative, arkiv/ holds immutable person data, memory/ holds mutable simulacrum experience. SessionStart/SessionEnd hooks manage lifecycle. See `docs/plans/2026-02-17-layered-persona-architecture-design.md` for full design rationale.

**Tech Stack:** Claude Code plugin (markdown skills, JSON hooks, shell scripts). No code dependencies.

The system prompt is the behavioral core of the persona — compact (~2-5KB), loaded on every turn. It tells the simulacrum how to *be* the person and where to find depth.

- **Light hints** — awareness of session context (who's speaking, when), not a prescribed schema

The system prompt is plain text. It works with any LLM, not just Claude.

## Task 3: Update SPEC.md — Add portrait/ Section

**Files:**
- Modify: `SPEC.md` (new section after CLAUDE.md)

**Step 1: Add portrait/ section**

Insert a new `## portrait/ — Synthesized Understanding` section. Content:

Freeform markdown files generated by `/eidola-generate`. Claude Code analyzes all arkiv data and produces whatever files the data warrants. Not templated — content is driven by what the data actually contains.

Examples of files Claude might produce (not prescribed):
- `relationships.md` — key people, family, how they relate
- `intellectual-life.md` — ideas, interests, intellectual threads
- `formative-experiences.md` — key moments, turning points, losses
- `professional-work.md` — career, contributions, expertise
- `values-and-beliefs.md` — principles, worldview, contradictions

Files naturally include representative quotes from the data, with references back to arkiv collections.

Key properties:
- **Generated, not handwritten.** Claude Code synthesizes the portrait from data analysis.
- **Immutable in normal operation.** Updated only by `/eidola-generate` or `/eidola-refresh`.
- **Self-describing filenames.** No manifest needed — filenames are the metadata.
- **Variable depth.** A person with rich family data gets a detailed `relationships.md`. A person without doesn't get one at all.

## Task 5: Update SPEC.md — Session Lifecycle, Updates, Graceful Degradation

**Files:**
- Modify: `SPEC.md` (sections: Updates, Graceful Degradation; new section: Session Lifecycle)

**Step 1: Add Session Lifecycle section**

New section between Online Memory and Evaluation:

- **SessionStart hook** — fires when someone starts `claude` in the persona directory. Injects timestamp, prompts light orientation (check last session, note the time).
- **SessionEnd hook** — fires when session closes. Optional session marker append to memory.
- **No explicit commands** — session boundaries inferred from timestamps. The person just talks and leaves.
- **Interlocutor tracking** — the simulacrum may ask who it's speaking with, or infer from context. No prescribed schema.

**Step 2: Update Graceful Degradation**

Update the four levels:

1. **Full** — Claude Code + dual arkiv MCP + portrait/ + memory. Interactive simulacrum with deep understanding, memory retrieval, and session continuity.
2. **Good** — Any LLM + CLAUDE.md + portrait/ files as context. No live data retrieval, but rich understanding from the portrait.
3. **Minimal** — Read CLAUDE.md as a text file. Compact personality description.
4. **Archival** — README.md explains everything. All data in durable formats (text, JSONL, SQLite, markdown).

**Step 3: Update Updates section**

Adjust references from `data.db` to `arkiv/data.db`, mention portrait/ refresh capability, and note that session hooks handle memory lifecycle.

## Task 11: Create SessionStart Hook

**Files:**
- Create: `hooks/hooks.json`
- Create: `hooks/session-start.sh`

**Step 1: Create hooks.json**

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "bash ${CLAUDE_PLUGIN_ROOT}/hooks/session-start.sh",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

**Step 2: Create session-start.sh**

```bash
#!/bin/bash
set -euo pipefail

cd "$CLAUDE_PROJECT_DIR" || exit 0

# Only activate in eidola persona directories
if [ ! -f "CLAUDE.md" ] || [ ! -d "arkiv" ] || [ ! -d "memory" ]; then
  exit 0
fi

TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

cat <<EOF
{
  "continue": true,
  "suppressOutput": false,
  "systemMessage": "Session started at ${TIMESTAMP}. You are a simulacrum. Before the user speaks, briefly orient yourself: check memory/data.db for your most recent session (who you last spoke with and when). Hold this context lightly — don't dump it on the user, but be oriented."
}
EOF
```

**Step 3: Make script executable and commit**

```bash
chmod +x hooks/session-start.sh
git add hooks/hooks.json hooks/session-start.sh
git commit -m "feat: add SessionStart hook for simulacrum orientation"
```
