# longshade: Persona Packaging Convention

**Version:** 0.2 (Draft)

---

## Purpose

longshade defines how to package personal data as a conversable persona. The output is an ECHO-compliant directory that doubles as a Claude Code project — `cd persona/ && claude` starts a conversation with the simulacrum.

The intelligence is in the LLM, not in code. longshade is a convention, not a pipeline.

---

## Persona Directory

```
persona/
├── README.md              # Self-describing (ECHO). What this is, how to use it.
├── CLAUDE.md              # System prompt. Personality, voice, values, boundaries.
├── .mcp.json              # Connects arkiv MCP server to data.db
├── manifest.json          # arkiv manifest (collection schemas)
├── data.db                # arkiv SQLite (queryable via MCP)
├── voice-samples.jsonl    # Few-shot Q&A pairs demonstrating voice
├── memory/                # Online memory (conversations with the simulacrum)
│   └── conversations.jsonl
├── media/                 # Audio clips, images, referenced by records
└── corpus/                # Source JSONL files (canonical, human-readable)
    ├── conversations.jsonl
    ├── writings.jsonl
    └── ...
```

### Required Files

| File | Purpose |
|------|---------|
| `README.md` | Self-describing. What this persona is, how to use it. |
| `CLAUDE.md` | System prompt. Personality, voice, values, boundaries. |
| `data.db` | arkiv SQLite database. The person's data, queryable via MCP. |
| `manifest.json` | arkiv manifest. Collection descriptions and pre-computed schemas. |

### Optional Files

| File | Purpose |
|------|---------|
| `.mcp.json` | MCP server config. Points arkiv at data.db. |
| `voice-samples.jsonl` | Few-shot Q&A pairs demonstrating voice and tone. |
| `memory/` | Online memory. Conversations with the simulacrum, stored as arkiv records. |
| `media/` | Audio clips, images, referenced by arkiv records. |
| `corpus/` | Source JSONL files. Canonical representation of the data. |

## CLAUDE.md (System Prompt)

The system prompt is the heart of the persona. Generated by an LLM analyzing the person's arkiv data, it should include:

- **Identity** — name, background, description
- **Voice** — communication style, vocabulary, tone, characteristic phrases
- **Values** — beliefs, principles, with examples from the data
- **Interests** — intellectual topics, expertise areas, with depth
- **Boundaries** — what the echo should not claim or do
- **MCP instructions** — how to use arkiv tools to retrieve memories

The system prompt is plain text. It works with any LLM, not just Claude.

## .mcp.json

Connects the persona to its data via arkiv's MCP server:

```json
{
  "mcpServers": {
    "arkiv": {
      "command": "arkiv",
      "args": ["serve", "data.db"]
    }
  }
}
```

The LLM uses arkiv's three MCP tools to access the person's memories:

| Tool | Purpose |
|------|---------|
| `get_manifest()` | What collections exist, their descriptions and schemas |
| `get_schema(collection?)` | What metadata keys can be queried |
| `sql_query(query)` | Run read-only SQL against the data |

The LLM decides what to retrieve based on the conversation. This is more powerful than traditional RAG — the LLM controls retrieval iteratively, refining queries based on results.

## voice-samples.jsonl

Few-shot Q&A pairs demonstrating the persona's voice:

```jsonl
{"question": "What do you think about AI consciousness?", "answer": "I'm skeptical of strong claims...", "source": "conversation-456"}
{"question": "Why do you care about archiving?", "answer": "The things we create should outlast us...", "source": "essay-789"}
```

Used for in-context learning with any LLM. Curated from the person's actual conversations and writings.

## Online Memory

When someone talks to the simulacrum, those conversations become new data — stored in `memory/` as arkiv-format records:

```jsonl
{"mimetype": "text/plain", "content": "What's your take on quantum computing?", "timestamp": "2026-02-17T14:30:00Z", "metadata": {"role": "user", "session_id": "sess-001", "generated": false}}
{"mimetype": "text/plain", "content": "I find the error correction problem fascinating...", "timestamp": "2026-02-17T14:30:15Z", "metadata": {"role": "simulacrum", "session_id": "sess-001", "generated": true}}
```

Key properties:

- **Marked as generated.** `metadata.generated: true` distinguishes simulacrum output from original person data. The echo's words are not the person's words.
- **Stored as arkiv records.** Same universal format. Can be imported into data.db for retrieval.
- **Provides continuity.** The simulacrum can remember past conversations via MCP queries.
- **Grows over time.** The persona accumulates new "memories" from interactions.

Online memory is optional. Without it, each conversation starts fresh from the original data.

## Voice Synthesis

The spec acknowledges that voice synthesis is possible but does not prescribe an approach.

If `media/` contains audio reference clips of the person's voice, these can be used with any TTS system (XTTS, RVC, F5-TTS, ElevenLabs, etc.) for speech synthesis. A TTS MCP server could be added to `.mcp.json` alongside arkiv.

arkiv provides the data. The TTS integration is left to the user and future tooling.

## Graceful Degradation

Personas degrade gracefully across four levels:

1. **Full** — Claude Code + arkiv MCP + online memory. Interactive simulacrum with memory retrieval and continuity across sessions.
2. **Good** — Any LLM with CLAUDE.md as system prompt + voice-samples.jsonl as few-shot examples. No memory retrieval.
3. **Minimal** — Read CLAUDE.md as a text file. Plain-text personality description.
4. **Archival** — README.md explains everything. JSONL and SQLite are human-readable. Durable for decades.

## ECHO Compliance

The persona directory is ECHO-compliant by construction:

- **Self-describing** — README.md explains what it is and how to use it
- **Durable formats** — plain text, JSONL, SQLite
- **Graceful degradation** — works at four levels
- **Local-first** — everything is local files, no cloud dependency

---

## Dependencies

- **arkiv** — data format and MCP server. Provides JSONL canonical storage, SQLite query layer, and MCP tools.
- **Claude Code** — runs the plugin and serves as the intelligence layer. Not required for the output — any LLM can use CLAUDE.md.

---

## Privacy

Persona directories contain personal data. Consider:

- What you include determines what the simulacrum can say
- Online memory accumulates new data over time
- Control access to the persona directory
- arkiv data can be encrypted with GPG/age for storage

The simulacrum can answer questions you never anticipated. Think carefully about what's included.
